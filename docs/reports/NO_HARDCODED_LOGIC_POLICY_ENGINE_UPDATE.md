# No-Hardcoded Logic Policy Engine Update

## Goal
Remove customer-flow hardcoded logic from the CRM/AI path while keeping UI labels and translations allowed.

## Applied changes
- Ticket flow decisions are now handled by an AI structured JSON classifier, not Arabic/English regex rules.
- Ticket creation requires the configured policy fields before creating the official ticket.
- If the customer switches context during a pending ticket flow, the pending flow is paused silently and the current business question is answered first.
- Customer-facing ticket field collection and ticket confirmation remain AI-generated from runtime context and bot settings.
- Legacy ticket intent regex now returns no direct ticket creation and delegates to the AI policy engine.
- Legacy handoff keyword detection now returns no direct decision and delegates to CRM policy flow.
- Knowledge/business intent keyword routing was neutralized to avoid language-specific hardcoded logic.
- Entity extraction no longer uses language-specific lexical fallback service names.
- Customer-visible legacy i18n system messages were disabled so ticket/handoff confirmations are generated by AI only.

## New files
- `src/config/crm-ticket-policy.json`
- `src/lib/crm/ticket-policy.ts`
- `src/lib/crm/ticket-ai-classifier.ts`

## Important note
UI copy, dashboard labels, enum values, database categories, and admin translations remain allowed. The removed hardcoding is the logic that decides customer intent, ticket creation, field collection, and context switching based on fixed Arabic/English phrases.
